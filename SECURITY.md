# Security Policy

## Supported Versions

VOOM is pre-1.0 software. Security fixes are applied to the default branch and
included in the next tagged release. Older tags are not maintained unless a
release note explicitly says otherwise.

## Reporting a Vulnerability

Please report suspected vulnerabilities through GitHub private vulnerability
reporting for this repository:

https://github.com/randomparity/voom/security/advisories/new

Do not open a public issue for a vulnerability before it has been triaged. When
reporting, include the affected version or commit, the impact, reproduction
steps, and any relevant logs or proof-of-concept input.

## Response Expectations

Maintainers will acknowledge valid reports as time permits, triage the impact,
and coordinate a fix or disclosure through the advisory. If a report does not
describe a security boundary or exploitable behavior, it may be redirected to a
normal GitHub issue.
